# Opik 多用户多工作空间认证鉴权技术方案

## 文档概览

本技术方案旨在为 Opik 开源版本实现完整的多用户、多工作空间支持，包括认证鉴权、权限管理、审计日志等企业级功能。

## 版本信息

- **文档版本**: v1.0
- **创建日期**: 2025-01-20
- **最后更新**: 2025-01-20
- **状态**: 待实施

## 目标

1. **实现多用户系统**: 支持用户注册、登录、用户管理
2. **实现多工作空间**: 支持工作空间创建、配置、成员管理
3. **本地化认证服务**: 实现 reactService 的本地版本，替代远程服务调用
4. **RBAC 权限系统**: 实现细粒度的角色权限控制（系统级、工作空间级、项目级）
5. **审计日志系统**: 记录所有关键操作，支持查询和分析
6. **平台管理界面**: 提供完整的管理后台，支持中英文切换

## 关键特性

### 用户管理
- ✅ 用户注册/登录/登出（无需邮箱验证）
- ✅ 密码管理（修改、重置）
- ✅ 用户信息管理
- ✅ 系统管理员角色

### 工作空间管理
- ✅ 工作空间 CRUD（无创建数量限制）
- ✅ 工作空间成员管理
- ✅ 工作空间配额管理（默认配额：10）
- ✅ 工作空间管理员角色

### 认证方式
- ✅ Session 认证（Cookie-based，Web UI）
- ✅ API Key 认证（Header-based，SDK/API）
- ✅ Redis 缓存优化性能

### 权限控制
- ✅ 三级权限：系统级、工作空间级、项目级
- ✅ 预置角色：System Admin、Workspace Admin、Developer、Viewer
- ✅ 自定义角色支持
- ✅ API Key 权限范围限制

### 审计日志
- ✅ 记录所有操作（包括查看操作）
- ✅ ClickHouse 存储，高性能查询
- ✅ 日志保留 90 天（可配置）
- ✅ 支持导出和分析

### 平台管理
- ✅ 用户管理界面
- ✅ 工作空间管理界面
- ✅ 角色权限管理界面
- ✅ 审计日志查询界面
- ✅ 系统设置界面
- ✅ 中英文国际化支持

## 技术栈

### 后端
- **框架**: Dropwizard 4.0.14
- **数据库**: MySQL 9.3 (状态数据) + ClickHouse 0.9.0 (分析数据 + 审计日志)
- **缓存**: Redis (Redisson 3.50.0)
- **依赖注入**: Guice
- **数据访问**: JDBI3
- **密码加密**: BCrypt
- **构建工具**: Maven

### 前端
- **框架**: React 18.3.1 + TypeScript 5.4.5
- **构建工具**: Vite 5.2.11
- **路由**: TanStack Router 1.36.3
- **状态管理**: Zustand 4.5.2
- **UI 组件**: Radix UI
- **样式**: Tailwind CSS 3.4.3
- **国际化**: 复用现有 i18n 方案

## 设计原则

1. **向后兼容**: 不破坏现有开源版本的单工作空间模式
2. **数据隔离**: 严格的工作空间数据隔离
3. **性能优先**: 使用 Redis 缓存，减少数据库查询
4. **安全第一**: 密码加密、Session 安全、API Key 安全
5. **可扩展性**: 支持自定义角色和权限
6. **可观测性**: 完整的审计日志
7. **用户友好**: 简洁的 UI，支持中英文切换
